# Contacts REST API (FastAPI + SQLite + JWT)

Цей застосунок є REST API для зберігання та управління контактами. Він підтримує повноцінний механізм реєстрації, автентифікації та авторизації за допомогою токенів JWT (`access_token` та `refresh_token`). Кожен користувач має доступ **виключно до власних контактів**.

В якості бази даних використовується **SQLite**, що зберігається локально у файлі `contacts.db` і не потребує запуску зовнішніх серверів чи Docker-контейнерів.

## 🚀 Функціонал
* Реєстрація користувачів з валідацією пошти (`EmailStr`) та хешуванням паролів (чистий `bcrypt`).
* Авторизація через вбудовану OAuth2-форму та генерація пари JWT токенів.
* Повний CRUD для контактів (Створення, Читання, Оновлення, Видалення) з прив'язкою до `user_id` власника.
* Пошук контактів за ім'ям, прізвищем або електронною поштою за допомогою Query-параметрів.
* Отримання списку контактів, у яких день народження у найближчі 7 днів.

## 🛠️ Інструкція із запуску

### Крок 1: Клонування проєкту та створення середовища
Переконайтеся, що ви перебуваєте в терміналі в папці `goit-web-hw-12`. Створіть та активуйте віртуальне середовище Python:

```bash
# Для Linux / macOS
python3 -m venv venv
source venv/bin/activate

# Для Windows (в командному рядку CMD)
python -m venv venv
source venv\Scripts\activate
```

### Крок 2: Встановлення залежностей
Встановіть усі необхідні бібліотеки:
```bash
pip install -r requirements.txt
```

### Крок 3: Налаштування файлу конфігурації `.env`
Створіть файл `.env` у корені проєкту (поруч із `main.py`) та згенеруйте унікальний ключ підпису:
```bash
printf "DATABASE_URL=sqlite:///./contacts.db\nALGORITHM=HS256\n" > .env
python -c 'import secrets; print(f"SECRET_KEY={secrets.token_hex(32)}")' >> .env
```

Не використовуйте приклад або спільний ключ у різних середовищах і не додавайте `.env` чи локальну базу даних до Git.

### Крок 4: Запуск FastAPI застосунку
Запустіть сервер розробки за допомогою Uvicorn:
```bash
uvicorn main:app --reload
```
При першому запуску сервер автоматично створить файл **`contacts.db`** у вашій папці та згенерує в ньому всі необхідні таблиці. 

Сервер буде доступний за адресою: `http://127.0.0.1:8000`

## 📝 Як користуватися та тестувати (Swagger UI)

1. Відкрийте у браузері інтерактивну документацію: **`http://127.0.0.1:8000`**
2. **Реєстрація:** Знайдіть ендпоінт `POST /api/auth/signup`, натисніть *Try it out* та зареєструйте нового користувача, вказавши робочий JSON з `email` та `password`.
3. **Логін:** Перейдіть до `POST /api/auth/login`. 
   * *Зверніть увагу:* За стандартом OAuth2 дані передаються як форма (`application/x-www-form-urlencoded`).
   * Введіть свій email у поле **`username`**.
   * Введіть пароль у поле **`password`**.
   * Натисніть **Execute**. У відповідь сервер надішле `access_token` та `refresh_token`.
4. **Авторизація запитів:** 
   * Скопіюйте отриманий рядок `access_token` (без лапок).
   * Вгорі сторінки Swagger натисніть кнопку **Authorize** (із зображенням замочка).
   * Вставте токен у поле і натисніть **Authorize**.
5. Тепер ви можете створювати, видаляти та шукати контакти в розділі `/api/contacts`. Вони будуть надійно ізольовані у базі даних та прив'язані саме до вашого профілю.
